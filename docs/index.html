指紋センサー — 出力文字列と登録の管理

指紋センサーを接続

USBまたはBluetooth経由で、指ごとの出力文字列と指紋登録を ブラウザから管理します。データはこのタブとデバイスの間だけで やり取りされ、外部には一切送信されません。

Chrome または Edge 89以上、かつsecure context(HTTPS・localhost・ローカルファイル)が必要です。Firefox/Safariでは動作しません。

指スロット

スロットを選んで、名前・出力文字列・Enter送信・指紋登録を設定します。

「文字列」と「指紋」の表示について

「文字列設定済み/未設定」は出力文字列がNVSに保存されて いるかどうか、「指紋登録済み/指紋未登録」はセンサー 本体に指紋テンプレートが保存されているかどうかを表す、 互いに独立した情報です(出力文字列だけ設定して指紋登録前、ということ もあり得ます)。指紋側は「一覧を更新」を押した時にセンサーへ問い合わせて 取得するため、直後は「確認中」と表示されることがあります。

保存済みの文字列が表示されない理由(書き込み専用)

この画面から保存済みの文字列そのものを読み出すことは できません(書き込み専用)。 このシリアルコンソールには認証がなく、USB物理接続さえできれば 誰でもコマンドを送れるため、値をそのまま返す設計にすると指紋 認証を経由せずに全部読み出せてしまいます。それを防ぐための 仕様です。保存は常に新しい値での上書きのみになります。

出力文字列の保存場所と暗号化について

センサー本体のフラッシュ(NVS)にのみ保存されます — このシリアル 接続以外でデバイスの外に出ることはなく、このリポジトリに 書き込まれることもありません。保存時はチップ固有のIDから導出した 鍵でAES暗号化されており、フラッシュを単純に読み出しただけでは 平文は分かりません(ただし基板に直接デバッグ接続して解析するような 相手までは防げません)。

取り扱い注意
出力文字列の削除 保存済みの出力文字列とEnter設定を消します。指紋テンプレート自体は残ります。
指紋テンプレートの削除 センサー本体に登録された指紋データ自体を完全に削除します(PS_DeleteChar、取り消せません)。
デバッグログ
